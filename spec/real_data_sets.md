Financial Risk (The Classic PSI Use Case)In these datasets, you typically use older data as your Base/Reference distribution ($P$) and newer data as your Current distribution ($Q$) to see if borrower profiles have drifted.Lending Club Loan Data (Kaggle): This is the gold standard for practicing PSI. It contains millions of loans issued over several years.How to use it: Train your base distribution on loans from 2014, and calculate the PSI against loans from 2017. You will see massive drift in income and debt-to-income ratios.Default of Credit Card Clients (UCI Machine Learning Repository): A classic tabular dataset from Taiwan containing default payments, demographic factors, and credit data.How to use it: You can split the data by age groups or education levels to measure the distributional distance between different demographics.2. Temporal & Environmental (Perfect for Covariance Drift)Since we just built a sophisticated multivariate PyTorch estimator, you want datasets where the relationships between variables change over time (like our Escenario 4).Bike Sharing Demand (UCI / Kaggle): Contains hourly rental data correlated with weather, temperature, and humidity.How to use it: Use "Summer" data as distribution $P$ and "Winter" data as distribution $Q$. The individual variables (temperature, rentals) drift, but more importantly, their covariance matrix completely changes depending on the season. Our neural network will catch this easily.Electricity Market Dataset (often found via river or OpenML): A famous dataset in concept drift research. It tracks electricity prices in Australia, which fluctuate wildly based on season, time of day, and market demand.3. The "Instant Download" Python LibrariesIf you don't want to deal with downloading CSVs and cleaning pandas dataframes, several Python libraries dedicated to model monitoring have drift datasets built right in. You can download them with one line of code:Evidently AI (evidently): This is an open-source ML monitoring library. They have built-in toy datasets specifically designed to exhibit data drift.Code: from evidently.datasets import bike_sharing_dataset (It comes pre-split into reference and current data).Scikit-Learn (sklearn.datasets): You can use standard datasets and artificially split them to create drift.Example: Load the fetch_california_housing dataset. Use houses in Northern California as distribution $P$, and houses in Southern California as distribution $Q$.
